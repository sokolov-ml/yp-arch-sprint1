# YP-Architect-sprint1

## Описание проекта

## Проектирование

При проектировании перехода на микрофронты были приняты следующие решения:

### Использование монорепозитория:

- в текущий момент не ожидается существенного роста количества разработчиков и частоты изменений

### Использование Webpack Module Federation:

- Все микрофронты разрабатываются на одном стеке
- Имеется возможность вынести крупные библиотеки (React) в общие зависимости
- Нет причин в текущий момент усложнять разработку написанием дополнительного кода для обеспечения совместной работы микрофронтов

## Структура

Фронтенд будет разделён на три микрофронта, включающие в себя следующие модули:

### host - главное приложение:

- Общие библиотеки
- Header
- Footer

### users - управление пользователями:

- Страница входа
- Страница регистрации
- Информация о пользователе
- Popup редактирования профиля
- Popup смены аватара

### gallery - управление контентом:

- Кнопка добавления карточки в галерею
- Popup добавления карточки
- Галерея
- Popup просмотра карточки

## Запуск

В зависимости от того, какое приложение вы хотите запустить, необходима разная последовательность действий:

### Монолитный frontend

- Перейти в каталог `./frontend/`
- Перед первым запуском не забыть установить зависимости:
  ```bash
  npm i
  ```
- Выполнить команду
  ```bash
  npm run start
  ```

### Новые микрофронты

Все микрофронты находятся в каталоге `./frontend/microfrontend/`

> Каждый микрофронт необходимо запускать в отдельном окне терминала. Микрофронт `host` должен запускаться в последнюю очердь

- Перейти в каталог микрофронта
- Перед первым запуском не забыть установить зависимости:
  ```bash
  npm i
  ```
- Выполнить команду
  ```bash
  npm run start
  ```
